# 高校社团平台 - 前后端文件说明

## 1. 项目目录

- `frontend/`: React 前端
- `backend/`: Flask 后端
- `docs/`: 项目文档
- `vercel.json`: Vercel 部署与路由配置

## 2. 前端文件（React）

### 核心入口
- `frontend/index.html`: 前端挂载点
- `frontend/src/main.jsx`: React 入口
- `frontend/src/App.jsx`: 全局导航与路由
- `frontend/src/styles.css`: 全局样式（高校平台风格）

### 页面
- `frontend/src/pages/HomePage.jsx`: 首页，包含引导语和弹框进入问卷
- `frontend/src/pages/QuestionnairePage.jsx`: 7步问卷与结果页
- `frontend/src/pages/ClubsPage.jsx`: 所有社团页面（搜索+卡片）
- `frontend/src/pages/SimilarPeoplePage.jsx`: 相似的人页面
- `frontend/src/pages/AIRecommendPage.jsx`: AI 推荐页面

### 组件与接口
- `frontend/src/components/StartModal.jsx`: 首页弹框组件
- `frontend/src/services/api.js`: 前端请求后端 API 封装

### 构建配置
- `frontend/package.json`: 前端依赖和脚本
- `frontend/vite.config.js`: 本地开发代理（/api -> Flask）

## 3. 后端文件（Flask）

### 核心
- `backend/app.py`: Flask 主应用，提供所有 API
- `backend/requirements.txt`: 后端依赖

### 数据与服务
- `backend/data/clubs.py`: 社团样例数据
- `backend/data/sample_users.py`: 相似用户样例数据
- `backend/services/storage.py`: Survey 存储层（优先 Supabase，失败回退内存）
- `backend/services/recommender.py`: AI 推荐逻辑（OpenAI 或规则引擎）

## 4. 前后端怎么连在一起

## 本地开发
1. 前端通过 `frontend/src/services/api.js` 访问 `/api/*`。
2. Vite 在 `frontend/vite.config.js` 配置了代理，将 `/api/*` 转发到 `http://127.0.0.1:5000`。
3. Flask 在 `backend/app.py` 提供：
   - `GET /api/clubs`
   - `POST /api/surveys`
   - `GET /api/similar-users`
   - `POST /api/ai-recommend`

## Vercel 线上部署
1. `vercel.json` 将 `/api/*` 路由到 Flask（`backend/app.py`）。
2. 其余路由返回 React 构建产物 `frontend/dist/index.html`。
3. React 页面里调用 `/api/*`，线上会自动落到 Flask 后端。

## 5. 数据流

1. 用户在问卷页面提交数据 -> `POST /api/surveys`
2. 后端写入 Supabase（配置后）或内存（未配置时）
3. 用户点击 AI 推荐 -> `POST /api/ai-recommend`
4. 后端读取用户问卷 + 社团标签 -> 生成推荐返回前端
5. 用户进入相似的人 -> `GET /api/similar-users`

## 6. 你可以优先改的地方

- 改问卷字段：`frontend/src/pages/QuestionnairePage.jsx`
- 改社团卡片内容：`backend/data/clubs.py`
- 改 AI 推荐提示词：`backend/services/recommender.py`
- 改导航结构：`frontend/src/App.jsx`
